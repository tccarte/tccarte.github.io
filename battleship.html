Battleship — Naval Command

A full-stack web Battleship game with a hunt/target AI opponent, local 2-player mode, persistent player stats, and a REST API built for automated grading.

▶ View Live App View on GitHub
PHP MySQL JavaScript REST API AJAX Sessions

What It Does

Players deploy a fleet of 5 ships on a 10×10 grid and take turns firing at the opponent's board to sink their fleet. The app supports two modes: single-player against a computer opponent using a hunt/target AI strategy, and local 2-player with handoff screens to keep each player's board hidden between turns.

Player statistics (wins, losses, total moves) are stored persistently in MySQL, and a leaderboard tracks all-time records. Game state is also saved to localStorage so a browser refresh doesn't lose an in-progress match.

Architecture

The frontend handles all UI, ship placement, and fires shots via AJAX. The backend is split into two layers — the game server manages session-based state and never exposes the computer's ship positions to the client, while a separate stateless REST API handles authenticated grading endpoints.

Browser ──AJAX──► game.php ──► $_SESSION + MySQL Grader ──HTTP──► api.php ──► MySQL

The key architecture decision was keeping all authoritative game state on the server in PHP sessions. The client only stores temporary UI state (what ship you're currently placing), which prevents any client-side cheating and keeps the AI's ship positions hidden until game over.

AI Strategy

The computer opponent uses a two-phase hunt/target algorithm. In hunt mode it fires randomly. Once it scores a hit it switches to target mode, adding all adjacent cells to a queue and systematically working through them until the ship sinks, then returns to hunt mode.
